## EXPLAIN

### 핵심 한 줄

세 번째 PR = 본인이 오늘 업무에 직접 적용한 1사이클의 증거다.

---

### 개념 풀이

**git log 3개 commit이 의미하는 것**

지금 본인 repo의 git log를 보면 3개의 commit이 쌓여 있다.

```
commit 3 — (오늘) 업무 1사이클 적용 결과
commit 2 — (어제) 두 번째 PR: Clarify + Deep Interview 반영
commit 1 — (Day 1) 첫 번째 PR: CLAUDE.md 초안
```

이 3개의 commit이 "내가 3일 동안 무엇을 바꿨는지"를 보여주는 타임라인이다. 코드가 없어도 된다. SKILL.md 한 줄 변경도 commit이다.

---

**변화량 시각화 — 짝꿍과 5분 공유**

git diff로 어제 버전과 오늘 버전을 비교하면 "어제는 이렇게 썼는데, 오늘은 이렇게 바꿨다"가 눈에 보인다.

짝꿍과 비교하는 포인트:

| 비교 항목 | 어제 | 오늘 |
|-----------|------|------|
| SKILL.md 길이 | ? 줄 | ? 줄 |
| 가장 크게 바뀐 섹션 | ? | ? |
| 오늘 새로 추가한 조건 | — | ? |

"길어졌다/짧아졌다"가 아니라 "무엇이 구체화됐는가"가 핵심이다.

---

**회고 1줄 — 내 암묵지의 단서**

하루 동안 스킬을 돌리면서 자주 수정하게 된 항목이 있다. 그게 바로 본인이 "당연하다고 여겨서 쓰지 않은" 암묵지다.

예시:
- "고객 이름을 빠뜨리는 걸 세 번 수정했다" → 고객 이름 명시가 내 기준에서 중요한 항목
- "톤을 계속 다듬었다" → 문체 조건이 SKILL.md에 충분히 기술되지 않았음

회고 1줄 형식: `"오늘 가장 자주 빠뜨린 항목: [항목명] — 이유: [한 줄]"`

이 1줄이 다음 개선 사이클의 시작점이다.

---

**Day 4 미리보기**

오늘 회고에서 나온 "가장 자주 빠뜨린 항목"이 Day 4의 재료가 된다.

Day 4에서는:
- 역할 분리 심화: 까다로운 고객 페르소나로 본인 스킬 결과를 검증
- Compound 사이클: 검증 결과를 반영해 SKILL.md를 자동으로 개선
- 팀 repo PR: 개인 스킬을 팀 공유 자산으로 올리기

오늘 회고 1줄이 Day 4 검증의 첫 번째 테스트 케이스가 된다.

## EXECUTE

### 지금 직접 해보세요

**1단계 — 변경사항 commit (5분)**

오늘 수정한 SKILL.md를 저장하고 commit한다.

```bash
git add SKILL.md
git commit -m "refine: [스킬 이름] - 오늘 업무 적용 후 보완"
```

commit 메시지에 "무엇을 보완했는지" 한 줄을 담는다.
(예: `refine: recap-mail - 고객 이름 + 톤 조건 추가`)

---

**2단계 — PR 생성 (3분)**

GitHub에서 PR을 생성한다. 리뷰어는 어제 짝꿍으로 지정한다.

PR 제목 형식: `[본인 이름] Day 3 스킬 개선 — [스킬 이름]`

PR 본문에 아래 3가지를 짧게 적는다:
1. 오늘 어떤 업무에 스킬을 적용했는가
2. 어제 대비 가장 크게 바뀐 부분은 무엇인가
3. 아직 미완성이거나 내일 다듬을 부분은 무엇인가

---

**3단계 — 짝꿍과 5분 공유**

짝꿍의 git log 3개를 같이 본다. 서로에게 물어볼 것:

- "이 commit에서 가장 많이 고민한 부분이 뭐야?"
- "어제 Clarify에서 나온 기준 중 오늘 실제로 써보니 어떻던?"
- "PR 본문에 쓴 '아직 미완성' 부분 — 내일 어떻게 다듬을 생각이야?"

짝꿍의 SKILL.md에서 "이건 나도 넣어야겠다"는 항목 1개를 메모해두면 이후 사이클에서 참고할 수 있다.

---

**4단계 — 회고 1줄 (3분)**

아래 형식으로 회고를 작성한다.

```
오늘 가장 자주 빠뜨린 항목: [항목명]
이유: [SKILL.md에 명시하지 않았거나 당연하다고 여겼기 때문에]
다음 수정 방향: [어떻게 추가/수정할지 한 줄]
```

이 회고는 SKILL.md 파일 하단 `## 회고` 섹션에 날짜와 함께 추가해둔다. Day 4에서 이 내용을 다시 꺼내 쓴다.
