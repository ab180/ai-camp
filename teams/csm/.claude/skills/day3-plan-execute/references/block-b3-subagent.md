## EXPLAIN

### Subagent = 독립 작업자를 동시에 여러 명 띄우기

```
일반 순차 방식:
  작업 A → 작업 B → 작업 C  (시간: 3배)

Subagent 병렬 방식:
  작업 A ──┐
  작업 B ──┼──► 동시에 처리 (시간: ~1배)
  작업 C ──┘
```

| 항목 | 설명 |
|------|------|
| **한 줄 정의** | 일회용 작업자를 동시에 여러 명 띄운다 — 결과는 메인이 합친다 |
| **왜 "일회성"** | Subagent는 그 작업에만 집중. 메인 대화의 다른 맥락에 오염되지 않음 |
| **왜 "병렬"** | 순차 3회 = 시간 3배. 병렬이면 거의 1배 |
| **언제** | 서로 결과를 참조하지 않는 독립 작업 2개 이상일 때 |

### CSM에서 쓸 수 있는 상황

```
예: 5개 고객 체크업 준비를 동시에
  → 각 고객 Subagent 5개 병렬 호출
  → 5개 결과를 메인이 표로 합침
  → 혼자 순차로 했을 때의 1/5 시간
```

### 주의점 3가지 (꼭 알아두기)

| 주의점 | 실제 문제 | 해결 |
|--------|---------|------|
| **맥락 공유 X** | Subagent는 메인 대화를 못 봄 | 필요한 맥락을 입력으로 명시해서 넘기기 |
| **비용 증폭** | 병렬 호출 = 토큰 합산. 5개면 5배 | 2~3개 범위에서 시작. 무조건 병렬 X |
| **합치기 책임** | 각 결과를 메인이 합쳐야 한 답이 됨 | "결과를 하나의 리캡 메일로 합쳐줘" 명시 |

## EXECUTE

```
아래 두 고객의 이번 주 중요 업무를 동시에 파악해줘.
각 고객을 독립적으로 조사해서 결과를 표로 합쳐줘.

고객 A: [이름]
고객 B: [이름]
