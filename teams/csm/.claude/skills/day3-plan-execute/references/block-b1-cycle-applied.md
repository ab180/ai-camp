## EXPLAIN

### 1. 핵심 한 줄

같은 사이클을 내 케이스에 2-3회 반복하면 "항상 빠뜨리는 항목의 패턴"이 보이고, 그것이 내 암묵지다.

---

### 2. 슬라이드 흐름 따라 개념 풀이

**운영진 케이스 → 본인 케이스**

Day 2에서는 운영진 케이스를 통해 사이클이 어떻게 돌아가는지 봤다. 오늘은 그 사이클을 내 손으로 직접 돌린다. 구경과 직접 실행은 체감이 다르다.

사이클은 동일하다:

```
자연어 부탁 → SKILL.md 읽기 → Clarify 4가지 → 재부탁
```

달라지는 것은 내용이다. 내 고객, 내 미팅, 내 언어로 채워진다.

**반복하면 무엇이 보이는가**

케이스 #1에서 사이클을 돌리면 Clarify 단계에서 "아, 이걸 안 말했구나"가 나온다. 그리고 케이스 #2에서 같은 사이클을 돌리면 — 종종 같은 항목이 또 빠진다.

예를 들어:

- 케이스 #1 리캡 메일: "출력 형태를 안 말했다" (불릿인지 단문 문단인지)
- 케이스 #2 온보딩 요약: "출력 형태를 또 안 말했다"

이것이 패턴이다. 매번 빠뜨리는 항목이 본인의 암묵지다. 머릿속에는 있는데, 글로 꺼내지 않는 것. 사이클이 그것을 드러낸다.

**70% → 90% 체감**

첫 번째 자연어 부탁에서 나오는 결과는 70%다. Clarify 후 재부탁하면 90%에 가까워진다. 그 간격이 작아지려면 사이클을 반복해야 한다. 한 번에 100%가 오지 않는다는 것이 전제다.

반복할수록 "처음 부탁할 때부터 어떤 항목을 포함해야 하는가"가 몸에 익는다. 그게 오늘이 끝날 때 가져가는 것이다.

**Day 2와의 차이**

| | Day 2 | Day 3 |
|---|---|---|
| 케이스 | 운영진 케이스 | 본인 케이스 |
| 역할 | 구경 | 직접 실행 |
| 목표 | 흐름 파악 | 암묵지 식별 |
| 반복 횟수 | 1회 | 2-3회 |

### 3. CSM 일상 적용

**케이스 #1 — 리캡 메일 예시**

자연어 부탁: "지난 [내가 고른 고객] 미팅 리캡 메일 초안 써줘"

Clarify 후 추가되는 것: 미팅 날짜, 참석자, 주요 논의점 3가지, 다음 액션과 담당자, 출력 형태(불릿 vs 단문 문단)

이 중에서 "출력 형태"는 처음 부탁할 때 대부분 빠진다. 두 번째 케이스에서도 또 빠지면 — 그것이 나의 암묵지다.

**케이스 #2 — 온보딩 현황 요약 예시**

자연어 부탁: "[내가 고른 고객] 온보딩 현황 요약해줘"

Clarify 후 추가되는 것: 요약 대상 기간, 리스크 기준(지연 며칠부터?), 수신자(팀장 vs 팀원), 출력 길이

여기서도 "수신자"와 "출력 길이"가 종종 빠진다. 케이스 #1에서 빠진 항목과 겹치면 그것이 패턴이다.

---

### 4. 흔한 오해

**오해 1 — "케이스마다 사이클이 달라야 한다"**

아니다. 사이클은 동일하다. 내용만 다르다. 같은 흐름을 반복하는 것이 오늘의 핵심이다.

**오해 2 — "Clarify 4가지는 정해진 질문이다"**

Clarify의 4가지 질문은 매 케이스마다 달라진다. 리캡 메일에서는 "출력 형태"가 나올 수 있고, 온보딩 요약에서는 "리스크 기준"이 나올 수 있다. 공통으로 자주 나오는 것이 내 암묵지다.

**오해 3 — "케이스 3개를 다 완성해야 한다"**

시간 내에 2개를 완주하는 것이 3개를 절반씩 하는 것보다 낫다. 케이스 #1과 #2에서 패턴이 보이면 오늘 목표는 달성이다. 케이스 #3은 시간이 남으면 한다.

## EXECUTE

**지금 직접 실행해보세요**

### 케이스 #1 사이클 (15-20분)

**1단계 — 자연어로 스킬 부탁**

B0에서 고른 케이스 #1을 지금 Claude에게 말 그대로 부탁한다. 완벽하게 설명하려 하지 않아도 된다. 지금 머릿속에 있는 말로 그냥 입력한다.

```
예시:
"[내가 고른 업무 이름] 스킬 만들어줘"
또는
"[내가 고른 고객] 리캡 메일 초안 써줘"
```

결과를 받으면 멈추고 읽는다.

---

**2단계 — SKILL.md 열어서 훑어보기**

Claude가 만든 SKILL.md를 열고 아래 두 가지를 확인한다:

- 입력값으로 어떤 항목을 받고 있는가?
- 빠진 것이 보이는가?

빠진 것을 메모해둔다. (아직 재부탁하지 않아도 된다)

---

**3단계 — Clarify 4가지로 재질문**

SKILL.md를 읽고 나서 Claude에게 4가지를 질문한다:

```
1. 이 스킬에서 입력값으로 받는 것은 무엇인가?
2. 출력 형태(포맷)는 어떻게 돼 있는가?
3. 어떤 경우에 이 스킬을 쓰는가?
4. 내가 원하는 톤/어조가 반영돼 있는가?
```

이 중에서 답이 명확하지 않은 항목이 Clarify 대상이다. Claude에게 해당 항목을 구체화해달라고 요청한다.

---

**4단계 — 답을 모아 재부탁**

3단계에서 확인한 내용을 채워서 다시 부탁한다. 1단계 결과와 비교해본다. 무엇이 달라졌는가?

다른 점을 메모한다. 그것이 Clarify가 채운 것이다.

---

### 케이스 #2 사이클 (15-20분)

같은 흐름을 케이스 #2로 반복한다.

1단계 → 2단계 → 3단계 → 4단계

이번에는 3단계에서 "케이스 #1에서 빠졌던 항목이 여기서도 빠졌는가?"를 확인한다.

**체크포인트 — 2회 완료 후**

> 케이스 #1과 #2에서 공통으로 빠진 항목이 있는가?

있다면 그것이 내 암묵지다. 메모장에 한 줄로 적어둔다:

```
"나는 항상 [___]를 빠뜨린다"
```

---

### 케이스 #3 (시간 되면, 10-15분)

케이스 #3은 선택이다. 시간이 남으면 같은 흐름으로 한 번 더 돌린다.

확인할 것: 케이스 #1, #2보다 빠르게 진행되는가? 처음 부탁할 때부터 더 많은 항목을 포함하게 되는가?

빠뜨리는 항목이 줄어들고, 처음 결과가 80-85%에 가까워지면 사이클이 익어가고 있는 것이다.
