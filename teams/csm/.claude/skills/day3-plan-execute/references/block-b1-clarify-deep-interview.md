# Block B1: Clarify + Deep Interview로 암묵지 끌어내기

> **Phase A 시작 시 반드시 아래 형태로 출력한다:**
> ```
> 📖 공식 문서: https://code.claude.com/docs/ko/skills
> ```

## EXPLAIN

### Clarify = "원하는 것"을 말로 만드는 단계

```
"AI가 바로 실행하기 전에
 정확히 무엇을 만들까를 되묻는 절차"
```

| 항목 | 설명 |
|------|------|
| **왜 필요** | "리포트 써줘"라고 말할 때 내 머릿속엔 2~3개 버전이 있다. 묻지 않으면 AI가 그중 하나를 무작위로 고른다 |
| **CSM 예시** | "이탈 위험 리포트" + Clarify → 독자(영업팀장), 핵심 질문 3개, 결론 먼저 — 이 3가지가 명시된 뒤에야 쓸만한 리포트가 나온다 |
| **질문이 느린 게 아님** | 잘못된 답을 다시 만드는 시간보다 Clarify가 짧다 |

### Deep Interview = Clarify를 구조로 박은 스킬

즉흥 Clarify보다 빠짐이 적다.

```
문제 1줄 입력 → 단계별 질문 → 스펙 문서 산출

반드시 묻는 4가지:
  독자   — 누가 읽나?
  입력   — 무엇을 넣으면?
  출력   — 어떤 형태로 나오면?
  성공 기준 — 언제 "됐다"인가?
```

**왜 중요**
재사용 스펙 문서가 남는다. 일회성 대화가 아니라 자산.
이 스펙 문서가 다음 블록(B2)의 SKILL.md 다듬기 입력이 된다.

### 4단계 실습 흐름

```
1/4 문제 고르기 — Day 2에서 느낀 답답함 1개 선택 (5분)
2/4 Deep Interview 호출 (10분)
3/4 스펙 받기 — 독자·입출력·성공기준 확인 (10분)
4/4 짝꿍 점검 — 스펙 서로 읽어보기 (5분)
```

## EXECUTE

**1단계: 문제 1개 고르기 (5분)**

어제 B0에서 이름 붙인 빠진 항목 중 1개 선택. 또는:
```
Day 2 스킬을 호출했을 때 가장 답답했던 점 1개를 적어줘.
구체적으로: "[이 부분이] 빠졌다" 또는 "[이 표현이] 어색했다" 형태로.
```

**2단계: Deep Interview 스킬 호출 (10분)**

Claude Code에서:
```
/deep-interview
```
> Deep Interview 스킬이 없다면: "내 스킬을 다듬고 싶어. 어떤 부분을 채워야 할지 질문해줘"라고 말로 진행.

**3단계: 받은 스펙 읽고 확인 (10분)**

- 독자가 명확한가? ("영업팀장" vs "팀 내부")
- 입력 조건이 구체적인가? (고객명·기간 등)
- 출력 형태가 그려지는가? (bullet 3개 vs 한 단락)
- 성공 기준이 있는가? ("이거면 됐다"를 말할 수 있나)

```json
AskUserQuestion({
  "questions": [{
    "question": "Deep Interview에서 반드시 묻는 4가지가 아닌 것은?",

      {"label": "비용 — 얼마나 걸리나", "description": "Deep Interview 필수 항목 아님"},
      {"label": "입력 — 무엇을 넣으면", "description": "반드시 포함"},
      {"label": "성공 기준 — 언제 됐다인가", "description": "반드시 포함"}
    ],
    "multiSelect": false
  }]
})
```

정답: 비용 — 얼마나 걸리나.
해설: Deep Interview의 필수 4가지는 독자·입력·출력·성공 기준입니다. 비용이나 일정은 포함되지 않습니다. 이 4가지가 명확해야 AI가 "정확히 무엇을 만들어야 하는지"를 알 수 있습니다.
