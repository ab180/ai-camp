## EXPLAIN

### 어제 빠진 부분이 오늘 다듬을 항목이다

어제 호출 결과의 부족한 부분 = 머릿속엔 있었는데 SKILL.md엔 안 적힌 것.
그걸 먼저 언어로 바꿔야 SKILL.md에 채울 수 있다.

**3가지 관점으로 짚어보기**

| 관점 | 질문 | 스킬 후보 기준 |
|------|------|-------------|
| **반복** | 한 번만 있었나, 3번 이상 있었나? | 반복되는 답답함만 스킬 후보. 일회성은 이번만 잘하면 끝 |
| **구체성** | "애매하게 별로"인가, "X가 없었다"인가? | "그냥 별로"는 스킬로 못 만든다. "다음 액션 날짜가 빠졌다"처럼 항목이 나와야 |
| **내 손** | 지금 내가 수동으로 하고 있나? | 수동 작업이 있는 단계가 자동화 후보. 내 손이 안 움직이면 스킬이 해결할 게 없다 |

### 예시: 빠진 암묵지 이름 붙이기

```
어제 결과를 보며 발견한 것:
- "다음 분기 체크업 일정이 임의로 적혀 있었다"
  → 반복? O (매번 실수)
  → 구체성? O ("다음 분기 일정 임의 명시 금지"로 명확)
  → 내 손? O (내가 매번 지워야 함)
  → 스킬에 추가 대상

- "톤이 조금 딱딱했다"
  → 반복? 모르겠다
  → 구체성? X ("조금 딱딱"은 모호)
  → 지금은 보류. 더 구체적인 표현으로 바뀌면 그때 추가
```

## EXECUTE

지금 어제 SKILL.md 첫 호출 결과를 열어보고:

**짝꿍 공유 12분**
1. 어제 결과에서 "이 부분이 이상했다" 또는 "이게 빠진 것 같다"를 서로 1개씩 말하기
2. 3가지 관점(반복·구체성·내 손)으로 점검
3. "다음에 Claude Code에게 어떻게 말해야 이게 채워질까?" 같이 생각하기

혼자 진행 중이라면:
```
어제 내 SKILL.md 호출 결과에서 빠진 것이 있었나?
빠진 것이 있다면 어떤 항목인지 구체적으로 말해줘.
```
