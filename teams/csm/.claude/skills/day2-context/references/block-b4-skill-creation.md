# Block B4: 스킬 첫 제작 — 4 Step 분해 → 스펙 → SKILL.md 생성

> **Phase A 시작 시 반드시 아래 형태로 출력한다:**
> ```
> 📖 공식 문서: https://code.claude.com/docs/ko/skills
> ```

## EXPLAIN

### 통째로 시키면 스킬이 안 나온다

```
X "리캡 메일 잘 써줘"
  → 뭘 잘 쓰라는 건지 명확하지 않음
  → 일회성 프롬프트 → 재사용 불가

O 4 Step으로 쪼개기
  01 최근 미팅 노트 수집   입력: 고객명·기간 → 출력: Notion 노트 3-5개
  02 액션 아이템 추출      입력: 노트 텍스트 → 출력: 액션 bullet 3-5개
  03 고객 톤에 맞게 정리   입력: 액션 + CLAUDE.md 톤 → 출력: 메일 초안
  04 링크·첨부 자료 검증   입력: 초안 → 출력: 링크 점검 리포트

각 Step은 입출력이 명확하다. 이 중 하나만 스킬로 만들면 된다.
```

### 스킬 스펙 5줄 템플릿

고른 Step을 아래 5줄로 옮겨 적는다:

```
01 스킬 이름: __________________ (영문 소문자 + 하이픈. 예: extract-meeting-actions)
02 언제 쓰나: __________________ (한 문장. 예: "고객 리캡 메일 쓰기 전에")
03 입력:      __________________ (예: 고객명, 기간 (일수))
04 출력:      __________________ (예: 액션 아이템 bullet 3-5개)
05 사용 MCP:  __________________ (예: Notion MCP — 회의록 페이지 접근)
```

**채워진 예시**
```
01 스킬 이름: extract-meeting-actions
02 언제 쓰나: 고객 리캡 메일 쓰기 전
03 입력:      고객명, 기간 (14일)
04 출력:      액션 bullet 3-5개 + 원문 링크
05 사용 MCP:  Notion (고객별 회의록 폴더)
```

### Claude에게 스킬 만들게 시키기

스펙 5줄이 준비되면 Claude Code에서:
```
.claude/skills/extract-meeting-actions/SKILL.md 만들어줘.
스펙:
- 언제: 고객 리캡 메일 쓰기 전
- 입력: 고객명, 기간(일수)
- 출력: 액션 bullet 3-5개 + 원문 링크
- MCP: Notion
```

**첫 실행 결과는 70%만 쓸만하면 정상.**
스킬 다듬기는 내일 Day 3에서 Clarify + Deep Interview로.

## EXECUTE

**1단계: 내 반복 업무 1개 4 Step으로 쪼개기 (5분)**

Day 2 슬라이드의 분해 예시를 참고해서:
```
내 반복 업무: [예: 체크업 미팅 준비]

01 Step: 입력 → 출력:
02 Step: 입력 → 출력:
03 Step: 입력 → 출력:
04 Step: 입력 → 출력:
```

**2단계: 스킬로 만들 Step 1개 고르기 (2분)**

가장 자주 하는 단계 또는 가장 수동 작업이 많은 단계를 선택.

**3단계: 스펙 5줄 채우기 (5분)**

**4단계: Claude에게 SKILL.md 생성 시키기 (5분)**

Claude Code 세션에서 스펙을 붙여넣고 "만들어줘"라고 입력.

**5단계: 생성된 SKILL.md 한 번 호출해보기 (3분)**

```
/skill [스킬이름]
```

결과를 보고 "70%면 됐다"고 판단하면 완료.
