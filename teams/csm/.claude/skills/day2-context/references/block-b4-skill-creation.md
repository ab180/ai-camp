## EXPLAIN

### 1. 핵심 한 줄
스킬은 "매번 처음부터 설명하지 않아도 되는 반복 작업의 템플릿"이다. 4단계로 분해해서 만들고, 만들수록 내 일이 빨라지는 구조다.

---

### 2. 슬라이드 흐름 따라 개념 풀이

**스킬이란 무엇인가**

스킬은 반복되는 작업을 `/스킬이름`으로 호출할 수 있게 만든 템플릿이다. CLAUDE.md가 "항상 적용되는 기본값"이라면, 스킬은 "특정 작업을 할 때 불러오는 절차"다.

예시: 리캡 메일 스킬을 만들어두면 `/recap 고객A`만 입력해도 된다. CLAUDE.md의 톤·금지어 + 스킬의 절차 + MCP의 실시간 맥락이 함께 작동한다.

공식 문서 정의: *"스킬은 Claude가 다양한 유형의 작업을 수행하는 방법을 지정합니다. 스킬을 사용하면 Claude Code를 특정 작업에 맞춤화할 수 있습니다."*

**스킬 하나의 구조**

스킬은 보통 세 개의 파일로 구성된다.

```
skills/recap-mail/
├── SKILL.md       ← 스킬 정체성·호출 조건·대표 예시
├── templates/     ← 출력 포맷 (리캡 메일 본문 틀)
└── rules/         ← 톤·금지어·필수 섹션
```

오늘은 운영진이 만든 베이스 스킬(3종)을 받아서 내 맥락으로 수정한다. 처음부터 만드는 것이 아니라 "편집 모드"로 시작한다.

**4단계 분해 — 스킬 만드는 방법**

**Step 1 — Clarify (명확히 하기)**

스킬을 만들기 전에 먼저 묻는다. AI가 바로 실행하기 전에 "정확히 무엇을 만들까"를 확인하는 절차.

- 누가 이 스킬의 결과를 읽는가? (독자)
- 어떤 입력이 들어오는가? (미팅 노트? 고객 이름?)
- 어떤 출력이 나와야 하는가? (메일 초안? 불릿 요약?)
- 성공 기준은 무엇인가? ("이 정도면 보낼 수 있다"는 기준)

Clarify 없이 바로 만들면 "빠르게 틀린 스킬"이 나온다.

**Step 2 — Design (설계)**

Clarify 결과를 바탕으로 스킬의 구조를 잡는다. SKILL.md에 언제 쓰는 스킬인지, 무엇이 입력이고 출력인지를 명시한다.

**Step 3 — Customize (커스터마이징)**

운영진 베이스 스킬의 3가지를 내 맥락으로 바꾼다:
1. **담당 고객**: SKILL.md의 입력 섹션을 내 고객군으로 수정
2. **톤**: rules/tone.md를 내 고객 유형에 맞게 조정
3. **키워드**: rules/banned.md에 금지어, rules/required.md에 필수 표현 추가

이 3요소만 맞추면 베이스 스킬이 "내 것의 70%"까지 간다.

**Step 4 — Test (테스트)**

스킬을 한 번 실행해서 결과를 보고, Deep Interview 스펙과 대조한다. 70% 쓸만하면 합격. 나머지 30%는 쓰면서 채워간다.

**CLAUDE.md + 스킬 + MCP의 3층 구조**

```
CLAUDE.md        ← 기본값 (담당 고객·톤) — 항상 적용
스킬             ← 반복 작업의 절차 — 해당 작업 시 적용
MCP              ← 실시간 맥락 (지금 이 고객의 정보) — 자동 호출
```

오늘 커스터마이징은 "스킬 층"만 손댄다. 나머지 두 층은 Day 1~2에서 이미 쌓여있다.

---

### 3. CSM 일상 적용

**시나리오 1 — 리캡 메일 스킬**

미팅이 끝난 직후 `/recap 고객A`를 입력하면, 스킬이 CLAUDE.md의 톤·금지어를 적용하고 MCP로 지난 미팅 노트를 읽어서 초안을 만든다. 30초 안에 "보내도 될지" 판단하는 수준의 초안이 나온다.

**시나리오 2 — 문의 답변 스킬**

Plain 문의가 들어왔을 때 `/inquiry`로 스킬을 부르면 문의 본문을 입력받아 답변 초안을 만든다. 과거 유사 답변 참고, 레퍼런스 링크 포함, 금지어 회피가 자동으로 적용된다.

**시나리오 3 — 스킬을 팀이 함께 쓸 때**

내가 만든 스킬을 팀 repo에 올리면 팀원이 `clone`해서 바로 쓸 수 있다. 팀원이 개선하면 PR로 돌려받는다. 스킬이 팀 단위로 복리가 된다.

---

### 4. 흔한 오해

**오해 1 — "스킬은 개발자만 만들 수 있다"**

스킬 파일(SKILL.md, rules/, templates/)은 모두 텍스트 파일이다. 코드가 없다. "이 스킬은 리캡 메일을 만든다", "톤은 짧고 따뜻하게", "금지어: 반드시"처럼 한국어로 적으면 된다.

**오해 2 — "스킬이 있으면 CLAUDE.md가 필요 없다"**

스킬은 절차를 담고, CLAUDE.md는 기본값을 담는다. 둘이 함께 작동해야 최적이다. 스킬만 있고 CLAUDE.md가 없으면 스킬 결과도 "우리 팀 색깔"이 빠진 일반 결과가 나온다.

**오해 3 — "스킬을 처음부터 완벽하게 만들어야 한다"**

오늘 목표는 "돌아가는 상태"다. 70%만 쓸만해도 합격이다. 실제로 쓰면서 발견한 것을 고치면 된다. Clarify → Design → Customize → Test → 다시 Clarify의 반복이 스킬을 발전시킨다.

```

**Step 3: 3요소 수정 (30분)**

SKILL.md, rules/ 폴더를 열고:
- [ ] 담당 고객 섹션 수정 (내 고객군으로)
- [ ] 톤 규칙 조정 (엔터프라이즈인지 스타트업인지에 따라)
- [ ] 금지어·필수 키워드 추가

**Step 4: 한 번 실행**
```
claude
/skill [스킬이름]
```

결과를 보고 Deep Interview 스펙(B1에서 받은 것)과 대조한다.

**막혔을 때**:
- 스킬 호출이 안 된다 → SKILL.md 상단에 "언제 쓰는 스킬인지" 한 줄 명시 확인
- 톤이 반영 안 된다 → SKILL.md에 `rules/tone.md 참조` 명시 확인
- MCP가 안 불린다 → SKILL.md에 "이 스킬은 Notion MCP를 사용" 한 줄 추가
