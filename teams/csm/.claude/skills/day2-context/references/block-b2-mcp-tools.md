## EXPLAIN

### 1. 핵심 한 줄
MCP는 AI와 외부 도구를 잇는 표준 포트다. USB-C가 모든 기기를 하나의 규격으로 연결하듯, MCP는 어떤 AI든 어떤 도구든 표준 방식으로 연결한다.

---

### 2. 슬라이드 흐름 따라 개념 풀이

**CSM 도구 6개가 담는 맥락의 종류**

CSM 한 명의 하루에는 여러 도구가 있다. 각 도구는 서로 다른 종류의 맥락을 담는다.

| 도구 | 담는 맥락 | 변화 속도 |
|-----|---------|---------|
| Salesforce | 계약 단계·담당 영업·OPP 상태 | 느림 (주/월 단위) |
| Notion | 미팅 노트·로드맵·운영 자료 | 느림 (주 단위) |
| Plain | 문의·티켓·답변 히스토리 | 빠름 (하루 단위) |
| Gmail | 고객 메일 스레드·첨부·시간순 기록 | 빠름 (시간 단위) |
| Slack | 오늘의 결정·내부 논의·빠른 소통 | 매우 빠름 (분 단위) |
| Sheets | 지표·수치·주기 집계 | 느림 (주/월 단위) |

**왜 도구 맵부터 그리는가**

MCP를 아무거나 연결하지 않는다. 세 가지 이유가 있다.

1. **우선순위**: 내 업무에서 질문 빈도가 높은 도구 먼저. Plain·Gmail·Slack 등 빠른 맥락 도구부터.
2. **시간 비용**: MCP 연결은 1개당 5~15분. 전부 연결하면 실습 시간의 절반을 쓴다.
3. **신호 관리**: 너무 많으면 AI가 "어디서 읽을지" 혼동한다. 3개 이내로 시작.

**MCP = USB-C 비유**

MCP(Model Context Protocol) 전에는 도구마다 별도 연동 코드가 필요했다. Gmail용 코드, Notion용 코드, Slack용 코드가 각각 달랐다. AI가 바뀌면 전부 다시 만들어야 했다.

MCP 이후: 도구가 MCP 서버 하나를 제공하면 어떤 AI든 거기에 "꽂기"만 하면 된다. 한 번 만든 서버를 여러 AI가 재사용한다.

"USB-C 이전에는 충전기가 기기마다 달랐다. USB-C 이후에는 같은 케이블로 모든 기기를 충전한다." MCP가 그 표준 포트 역할을 AI와 도구 사이에서 한다.

공식 문서 정의: *"MCP는 모델이 다양한 도구와 상호작용할 수 있도록 하는 표준 프로토콜입니다. Claude는 이를 사용하여 파일 시스템, 데이터베이스, API 및 기타 외부 서비스와 상호작용합니다."*

**MCP 연결 3가지 방식 (개요)**

| 방식 | 난이도 | 대상 | 오늘 실습 |
|-----|------|-----|--------|
| claude.ai 웹 UI (OAuth) | 쉬움 | 비개발자 기본값 | O |
| 로컬 stdio 서버 | 어려움 | 개발자용 | 개념만 |
| 원격 HTTP/SSE | 어려움 | 기업 호스팅 | 개념만 |

오늘은 1번(claude.ai 웹 UI)으로만 실습한다.

**MCP 연결 전 주의사항**

Gmail MCP를 켜면 AI가 내 메일을 읽을 수 있다. 이는 편리하지만 권한 관리가 필요하다.

- **권한 범위**: 읽기만인지 쓰기(보내기)까지인지 OAuth 창에서 확인
- **토큰 저장**: claude.ai는 Anthropic 서버에 암호화 저장
- **해제 방법**: Settings에서 즉시 disconnect 가능

보안 사고의 대부분은 "연결했다는 것을 잊어서" 발생한다. 정기적으로 `/mcp` 명령으로 연결 목록을 점검하는 것이 좋다.

---

### 3. CSM 일상 적용

**시나리오 1 — "이 계약 언제 끝나?" 질문**

Salesforce MCP를 연결하면 "고객 A 계약 만료일은?"을 Salesforce를 직접 읽어서 답한다. Salesforce를 따로 열고 검색하는 과정이 사라진다.

**시나리오 2 — "지난 문의 뭐였지?" 질문**

Plain MCP를 연결하면 "고객 B 최근 문의 내용 요약해줘"가 Plain의 티켓을 직접 읽어서 요약한다.

**시나리오 3 — "팀에서 오늘 이 고객 관련 무슨 결정 났지?" 질문**

Slack MCP를 연결하면 해당 채널의 최근 논의를 AI가 직접 읽고 요약한다.

---

### 4. 흔한 오해

**오해 1 — "MCP를 연결하면 AI가 내 도구를 마음대로 수정할 수 있다"**

MCP 권한 범위를 설정할 수 있다. 읽기만 허용하면 AI가 읽기만 한다. 보내기(쓰기)는 별도로 허용해야 한다. 권한을 최소한으로 주는 것이 원칙이다.

**오해 2 — "MCP는 개발자가 설치해줘야 한다"**

오늘 실습하는 claude.ai 웹 방식은 클릭 4번으로 끝난다. 개발 지식이 필요 없다. 개발자가 필요한 것은 로컬 stdio나 원격 HTTP 방식뿐이다.

**오해 3 — "도구를 전부 연결해야 효과가 있다"**

오히려 너무 많이 연결하면 AI가 어디서 읽어야 할지 혼동하고 응답이 느려질 수 있다. 업무 빈도가 가장 높은 도구 1~2개로 시작하고 점차 늘리는 것이 좋다.
