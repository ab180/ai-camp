## EXPLAIN

### 1. 한 문장 요약
CLAUDE.md의 6개 섹션은 AI에게 "우리 팀 신입사원이 반드시 알아야 할 것"을 담는 칸이며, 섹션마다 다른 종류의 기본값을 설정한다.

---

### 2. 왜 필요한가 (Why)

**"아무거나 다 쓰면 된다"의 함정**

CLAUDE.md를 처음 만들면 "일단 생각나는 것 다 써볼게"라는 접근이 나오기 쉽다. 그러면 중요한 것과 사소한 것이 뒤섞이고, 나중에 내용이 많아져도 정작 AI가 일관되게 활용하기 어렵다.

**구조가 있어야 채울 수 있다**

6개 섹션은 CSM 업무에서 AI가 반드시 알아야 할 정보를 논리적으로 분류한 것이다. 각 섹션이 서로 다른 역할을 한다. 구조가 없으면 중복되거나 빠뜨리는 일이 생긴다.

**이게 없으면 어떻게 되나**

섹션 구분 없이 작성된 CLAUDE.md는 시간이 지나면서 관리하기 어렵고, 팀원이 읽어도 뭐가 중요한지 파악하기 힘들다. 6섹션 구조는 나중에 팀으로 공유할 때도 읽기 쉽게 만든다.

---

### 3. 본질 (What)

**6개 섹션의 역할**

| 섹션 | 담는 것 | 왜 중요한가 |
|------|---------|-----------|
| **01 우리 팀** | 팀 역할·목표·주요 반복 업무 | AI가 "이 팀이 뭐 하는 팀인지" 안다 |
| **02 담당 고객** | 대표 고객 유형·계약 구조·주요 이슈 | AI가 "어떤 고객을 상대하는지" 안다 |
| **03 톤앤매너** | 고객 앞 언어·내부 소통 스타일 | AI 문장이 내 스타일처럼 들린다 |
| **04 금지어** | 쓰면 안 되는 표현 + 이유 | AI가 문제 될 말을 일관되게 회피한다 |
| **05 워크플로우** | 매주·매달 반복되는 업무 패턴 | AI가 업무 흐름을 이해하고 맞게 돕는다 |
| **06 도구** | 내가 쓰는 CSM 도구들 | AI가 어떤 도구를 연결하면 되는지 안다 |

**비유: 신입사원 온보딩 문서**

새 팀원이 처음 왔을 때 "이것만 알면 우리 팀에서 일할 수 있어"라고 주는 문서를 떠올려라. 팀 소개, 담당 고객 특성, 말투, 하면 안 되는 것, 반복 업무, 쓰는 도구 — 이 6가지면 기본이 잡힌다. CLAUDE.md의 6섹션이 그 역할이다.

---

### 4. 어떻게 작동하나 (How)

**섹션 1 — 우리 팀**

팀의 역할과 주요 목표, 반복 업무를 적는다. AI가 내가 어떤 맥락에서 일하는지 이해하는 기반이 된다.

```
## 우리 팀
- 역할: 국내 엔터프라이즈 고객 담당 CSM
- 주요 목표: 고객 성공·계약 유지·확장
- 반복 업무: 리캡 메일, 문의 응대, 체크업 미팅 준비
```

**섹션 2 — 담당 고객**

내가 주로 상대하는 고객 유형을 구체적으로 적는다. "대기업"이 아니라 "연간 계약·분기 QBR·트래킹 오차 이슈" 같은 식으로.

```
## 담당 고객
- 주요 유형: 국내 엔터프라이즈 모바일 앱 마케팅 팀
- 계약 구조: 연 단위 라이선스 + 분기 체크업
- 자주 나오는 이슈: 이벤트 트래킹 오차, 리포트 커스터마이즈, 온보딩 지연
```

**섹션 3 — 톤앤매너**

같은 정보도 어떻게 쓰느냐가 CSM 메일의 절반이다. "짧게", "따뜻하게", "다음 액션을 마지막에"처럼 구체적으로.

```
## 톤앤매너
- 고객 앞: 짧고 따뜻하게, 이름 호명, 다음 액션 항상 마지막 줄
- 금지 톤: 번역투 영어 느낌, 과장된 마케팅 표현
```

**섹션 4 — 금지어**

가장 중요한 규칙: 금지어는 반드시 "왜"와 함께 적어야 한다. 이유 없는 금지어는 AI가 상황에 따라 무시할 수 있다.

```
## 금지어
- "반드시": 고객 기대치를 과하게 만들어 이후 분쟁 위험
- "100% 보장": 기술적으로 불가능한 약속
- "개발팀이 일정상...": 고객 앞에서 내부 변명 — 신뢰 깎음
```

**섹션 5 — 워크플로우**

매주, 매달 반복되는 업무를 한 줄씩 정리해두면 AI가 그 맥락에서 도움을 준다.

```
## 워크플로우
- 매주 월요일: 지난 주 고객 문의 정리
- 미팅 후 24시간 내: 리캡 메일 발송
- 분기마다: 체크업 미팅 + 성과 요약
```

**섹션 6 — 도구**

내가 실제로 쓰는 CSM 도구와 각각의 역할을 적는다. Day 2에서 이 도구들을 MCP로 연결한다.

```
## 도구
- Salesforce: 고객 계약·파이프라인 관리
- Plain: 고객 문의·티켓
- Notion: 미팅 노트·내부 문서
- Gmail: 고객 메일
- Slack: 팀 내 실시간 소통
```

---

### 5. 흔한 오해 (Common Misconceptions)

**오해 1: "금지어만 많이 나열하면 좋다"**

금지어는 이유와 함께 있을 때만 효과가 있다. "이유 없는 금지어는 AI가 어길 수 있다"는 것이 핵심이다. 공식 문서도 "왜"를 함께 적도록 강조한다. 100개 금지어보다 이유가 명확한 5개가 낫다.

**오해 2: "톤앤매너는 나만 알면 된다"**

AI가 따를 톤앤매너는 AI가 이해할 수 있게 적어야 한다. "따뜻하게"만 적으면 AI마다 해석이 다르다. "짧은 문장, 이름 호명, 다음 액션 마지막 줄"처럼 행동으로 적어야 일관되게 동작한다.

**오해 3: "6섹션을 모두 완벽하게 채워야 시작한다"**

오늘 목표는 0.5판이다. 담당 고객 3줄, 톤앤매너 DO/DON'T 2가지씩, 금지어 3개면 충분하다. 나머지는 실제로 쓰면서 채운다.

---

### 6. CSM 일상 적용

**시나리오 1: 리캡 메일 요청**
섹션 2(담당 고객)와 섹션 3(톤)이 잘 채워져 있으면, "고객 A 리캡 메일 써줘"만 해도 AI가 그 고객의 맥락과 내 말투로 초안을 쓴다.

**시나리오 2: 금지어 실수 방지**
섹션 4(금지어)가 충실하면 AI가 "반드시"나 "100% 보장"같은 표현을 쓰지 않는다. 피로가 없는 AI는 사람보다 금지어를 일관되게 지킨다.

**시나리오 3: 체크업 미팅 준비**
섹션 5(워크플로우)에 "체크업 미팅 전 준비 항목"을 적어두면, AI가 그 형식에 맞춰 미팅 자료를 구성해준다.

**단계 1: Claude에게 말로 시키기**
Claude Code에서 아래처럼 입력한다:
```
CLAUDE.md에 아래 6개 섹션을 추가해줘:

3. 톤앤매너: [내가 쓰는 메일 스타일]
4. 금지어: [절대 쓰면 안 되는 표현]
5. 워크플로우: [매주 반복하는 업무]
6. 도구: [내가 쓰는 CSM 도구들]
```

**단계 2: 필수 항목 확인**
- 담당 고객 3줄 이상
- 톤앤매너 DO 2가지 / DON'T 2가지
- 금지어 3가지 이상 (각각 이유 포함)

**확인**: 짝꿍에게 내 CLAUDE.md를 읽혀보고 "이 사람의 업무가 구체적으로 떠오르냐"고 물어본다.

**막혔을 때**:
- 어떻게 써야 할지 모르겠으면 → Claude에게 "예시 보여줘"라고 하면 된다
- 섹션이 너무 길어지면 → 핵심 2-3줄만 남기고 나머지 삭제
