## EXPLAIN

### 1. 핵심 한 줄
`/init`은 "빈 종이 공포"를 없애주는 도구다. Claude Code가 현재 폴더를 스캔해 CLAUDE.md 초안을 자동으로 만들어준다.

---

### 2. 슬라이드 흐름 따라 개념 풀이

**왜 초안이 필요한가**

CLAUDE.md를 만들어야 한다는 것은 알겠는데, 막상 빈 파일 앞에 앉으면 어디서부터 시작할지 모른다. 섹션 이름은 뭐고, 어떤 형식이고, 무엇이 꼭 있어야 하는지 — 처음 접하는 사람에게는 진입 장벽이다.

"내 CLAUDE.md를 새로 만들어야 한다"와 "이 초안을 내 것으로 다듬어야 한다"는 심리적으로 완전히 다르다. **편집 모드가 창작 모드보다 훨씬 빠르다.** 초안이 있으면 "이건 맞아, 이건 아니야, 이건 추가해야겠어"로 바로 판단할 수 있다.

**`/init`이 하는 일**

Claude Code 세션 안에서 `/init`을 입력하면 Claude가 현재 폴더를 스캔한다. 폴더에 기존 파일이 있으면 그것을 참고해 더 구체적인 초안을 만들고, 비어있으면 일반 구조로 만든다. 결과로 CLAUDE.md 파일이 생성된다. 섹션 구조와 예시 내용이 담긴 "뼈대"다.

공식 문서 정의: *"/init을 실행하여 시작 CLAUDE.md를 자동으로 생성합니다. Claude는 코드베이스를 분석하고 발견한 내용이 포함된 파일을 만듭니다. CLAUDE.md가 이미 존재하면 /init은 덮어쓰지 않고 개선 사항을 제안합니다."*

중요한 점: **이미 CLAUDE.md가 있어도** /init을 실행할 수 있다. 덮어쓰지 않고 추가할 섹션 아이디어를 제안한다.

**비유: 입사 첫날의 매뉴얼 초안**

새 팀원에게 "우리 팀 업무 매뉴얼 처음부터 만들어"라고 하면 막막하다. 하지만 "이 초안 보고 우리 팀에 맞게 수정해"라고 하면 훨씬 빠르다. `/init`이 그 초안을 만들어주는 역할이다.

```
/init 없이:
  빈 파일 → 뭘 써야 하지? → 검색 → 형식 파악 → 작성 (30분+)

/init 있으면:
  /init 실행 → 초안 생성 → 내 것으로 편집 (10분)
```

**주의: /init은 출발점이다**

생성된 초안에는 개발 관련 내용이 섞여 있을 수 있다. 그것은 /init이 고장난 것이 아니라, Claude Code가 원래 개발자 도구이기 때문이다. CSM 업무에 맞지 않는 내용은 지우고, 내 실제 업무(고객 유형, 톤, 금지어)로 채우는 것이 B3 실습에서 할 일이다.

---

### 3. CSM 일상 적용

**시나리오 1 — CLAUDE.md 처음 시작할 때**

CSM 업무 폴더를 새로 만들고, `/init`으로 구조의 뼈대를 잡는다. 그 뒤 오늘 B3에서 6개 섹션을 채운다. 처음부터 혼자 만드는 것보다 훨씬 빠르게 시작할 수 있다.

**시나리오 2 — 기존 CLAUDE.md를 업그레이드할 때**

몇 달 동안 쓰다 보면 "뭔가 빠진 것 같다"는 느낌이 든다. 이때 `/init`을 다시 실행하면 현재 파일과 폴더를 보고 추가하면 좋을 섹션 아이디어를 제안해준다.

**시나리오 3 — 팀 CLAUDE.md 초안 만들기**

팀 단위로 공용 CLAUDE.md를 처음 만들 때, 한 명이 `/init`으로 뼈대를 잡고 팀이 함께 채우는 방식이 효율적이다.

---

### 4. 흔한 오해

**오해 1 — "/init 하면 CLAUDE.md가 완성된다"**

/init은 구조의 뼈대(섹션 제목·예시 형식)를 잡아준다. 실제 내용인 내 담당 고객, 우리 팀 톤, 금지어 목록은 내가 직접 채워야 한다. 초안은 출발점이지 완성본이 아니다.

**오해 2 — "폴더가 비어있으면 /init이 안 된다"**

빈 폴더에서도 /init은 동작한다. 기존 파일이 없으면 일반적인 CSM/비즈니스 구조로 초안을 만든다. 오히려 아무것도 없는 상태에서 시작하기 좋다.

**오해 3 — "/init은 한 번만 할 수 있다"**

CLAUDE.md가 이미 있어도 언제든 `/init`을 다시 실행할 수 있다. 덮어쓰지 않고 개선 사항이나 추가 섹션을 제안한다. 주기적으로 실행해서 새로운 아이디어를 얻을 수 있다.

## EXECUTE

### 지금 직접 해보세요

**준비물**: Claude Code가 설치된 상태, 터미널(Mac: 검은 화면의 앱)

**단계 1: 폴더 만들기**
바탕화면이나 원하는 위치에 `csm-claude`라는 폴더를 만든다. (이름은 무엇이든 관계없다)

화면이 어떻게 보여야 하는가: Finder에서 새 폴더가 생성된 것이 보인다.

**단계 2: 터미널에서 폴더로 이동**
```
cd ~/Desktop/csm-claude
```
(바탕화면이 아닌 다른 위치에 만들었다면 그 경로로 변경)

막히면: `cd`는 "change directory"의 약자다. 폴더 안으로 들어가는 명령이다. `pwd`를 입력하면 현재 위치를 확인할 수 있다.

**단계 3: Claude Code 세션 시작**
```
claude
```
처음 실행이면 로그인 과정이 있다. claude.ai 계정으로 로그인.

화면이 어떻게 보여야 하는가: `>` 또는 `(You)` 같은 입력 대기 상태가 나타난다.

**단계 4: /init 실행**
Claude Code 안에서:
```
/init
```
입력 후 엔터. Claude가 폴더를 스캔하고 CLAUDE.md를 만든다.

화면이 어떻게 보여야 하는가: "CLAUDE.md를 생성했습니다" 또는 그와 유사한 메시지가 나온다. 처리 시간은 10초~1분 내외.

**단계 5: 확인**
Finder에서 폴더를 열거나, 터미널에서 `ls`를 입력해 CLAUDE.md 파일이 생겼는지 확인.

**막혔을 때**:
- `"claude: command not found"` → Claude Code가 설치되지 않음. 운영진에게 설치 방법 요청
- 로그인 창이 뜨면 → claude.ai 계정으로 로그인
- /init 후 파일이 안 보이면 → `pwd` 입력해 현재 위치 확인. 예상 폴더가 맞는지 체크
- 권한 오류가 나면 → 운영진 호출
