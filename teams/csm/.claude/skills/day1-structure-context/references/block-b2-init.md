## EXPLAIN

### 1. 한 문장 요약
`/init` 명령은 빈 종이 앞의 막막함을 없애주는 도구로, Claude Code가 현재 폴더를 스캔해서 CLAUDE.md 초안을 자동으로 만들어준다.

---

### 2. 왜 필요한가 (Why)

**"뭘 써야 할지 모르겠다"는 막막함**

CLAUDE.md를 만들어야 한다는 것은 알겠는데, 막상 빈 파일 앞에 앉으면 어디서부터 시작해야 할지 모른다. 섹션 제목은 뭐고, 어떤 형식으로 써야 하며, 무엇이 꼭 있어야 하는지 — 처음 접하는 사람에게는 진입 장벽이 된다.

**초안이 있으면 달라진다**

"내 CLAUDE.md를 새로 만들어야 한다"와 "이 초안을 내 것으로 다듬어야 한다"는 심리적으로 완전히 다르다. 초안이 있으면 판단이 쉬워진다. "이건 맞아, 이건 아니야, 이건 추가해야겠어" — 편집 모드가 창작 모드보다 훨씬 빠르다.

**이게 없으면 어떻게 되나**

처음부터 혼자 작성하려다 시간이 너무 걸리거나, 구조가 제각각이 되거나, 중요한 섹션을 빠뜨리는 일이 생긴다. /init은 이 진입 장벽을 없애는 역할이다.

---

### 3. 본질 (What)

**정확한 정의**

`/init`은 Claude Code의 내장 명령어다. 실행하면 Claude가 현재 폴더의 파일과 구조를 분석한 뒤, 그것을 바탕으로 CLAUDE.md 초안을 자동으로 생성한다.

공식 문서에 따르면: *"/init을 실행하여 시작 CLAUDE.md를 자동으로 생성합니다. Claude는 코드베이스를 분석하고 발견한 빌드 명령, 테스트 지침 및 프로젝트 규칙이 포함된 파일을 만듭니다. CLAUDE.md가 이미 존재하면 /init은 덮어쓰지 않고 개선 사항을 제안합니다."*

CSM 업무 폴더에서 실행하면 개발 관련 내용보다는 우리가 채울 섹션 구조의 뼈대가 만들어진다.

**비유: 입사 첫날의 매뉴얼 초안**

새로 입사한 팀원에게 "우리 팀 업무 매뉴얼 만들어"라고 하면 막막하다. 하지만 "이 초안 보고 우리 팀 맞게 수정해"라고 하면 훨씬 빠르다. /init이 그 초안을 만들어주는 역할이다.

```
/init 없이:
  빈 파일 → 뭘 써야 하지? → 검색 → 형식 파악 → 작성 (30분+)

/init 있으면:
  /init 실행 → 초안 생성 → 내 것으로 편집 (10분)
```

**주의할 점**

/init의 결과는 출발점이지 완성본이 아니다. 생성된 초안을 그대로 쓰겠다는 게 아니라, 구조를 참고해서 내 맥락으로 채워나가는 것이 목적이다.

---

### 4. 어떻게 작동하나 (How)

**1단계: 폴더 준비**
내 CSM 업무용 폴더를 하나 만든다. 아무 위치라도 좋다.

**2단계: Claude Code 열기**
터미널에서 그 폴더로 이동한 뒤 `claude` 명령을 입력해 Claude Code 세션을 시작한다.

**3단계: /init 실행**
Claude Code 세션 안에서 `/init`을 입력하고 엔터를 누른다.

**4단계: Claude의 스캔**
Claude가 현재 폴더에 있는 파일들을 살펴본다. 기존 파일이 있으면 참고하고, 없으면 일반적인 구조로 초안을 만든다.

**5단계: CLAUDE.md 생성**
몇 초에서 1분 안에 CLAUDE.md 파일이 만들어진다. 섹션 구조와 예시 내용이 포함된 초안이다.

**6단계: 편집**
생성된 초안을 열어서 내 실제 업무와 고객에 맞게 채운다. B3 블록에서 6개 섹션을 채우는 실습을 한다.

---

### 5. 흔한 오해 (Common Misconceptions)

**오해 1: "/init 하면 다 완성된다"**

/init은 구조의 뼈대를 잡아준다. 실제 내용 — 내 담당 고객, 우리 팀 톤, 금지어 목록 — 은 내가 직접 채워야 한다. 초안은 출발점이지 완성본이 아니다.

**오해 2: "폴더에 아무것도 없으면 /init이 안 된다"**

빈 폴더에서도 /init은 동작한다. 기존 파일이 있으면 그것을 참고해서 더 구체적인 초안을 만들고, 없으면 일반 구조로 만든다.

**오해 3: "/init은 한 번만 할 수 있다"**

CLAUDE.md가 이미 있는 상태에서 `/init`을 실행하면 덮어쓰지 않고 개선 사항을 제안한다. 언제든 다시 실행해서 추가 제안을 받을 수 있다.

---

### 6. CSM 일상 적용

**시나리오 1: 첫 CLAUDE.md 만들기**
CSM 업무 폴더를 새로 만들고 /init으로 구조를 잡은 뒤, 담당 고객·톤·금지어를 채운다. 이 과정이 오늘 실습의 핵심이다.

**시나리오 2: 팀에 공유하기**
내 CLAUDE.md가 어느 정도 쌓이면, 팀 폴더에 올려서 다른 팀원들도 같은 구조로 시작할 수 있게 기반을 제공할 수 있다.

**시나리오 3: 정기 업데이트**
3개월마다 CLAUDE.md를 검토하고 업데이트할 때, /init을 다시 실행해서 추가할 섹션 아이디어를 받을 수 있다.

## EXECUTE

### 지금 직접 해보세요

**준비**: Claude Code가 설치되어 있고, 로그인된 상태. 터미널(Mac: 검은 화면의 앱) 준비.

**단계 1: 폴더 만들기**
Finder에서 바탕화면에 `csm-claude` 폴더를 만든다 (이름은 아무거나).

**단계 2: 터미널에서 폴더로 이동**
```
cd ~/Desktop/csm-claude
```

**단계 3: Claude Code 시작**
```
claude
```
이 명령을 입력하면 Claude Code 세션이 시작된다. 처음 실행이면 로그인 과정이 있다.

**단계 4: /init 실행**
Claude Code 안에서 아래를 입력하고 엔터:
```
/init
```

**확인**: 완료되면 CLAUDE.md 파일이 생성되었는지 확인한다. `ls` 명령이나 Finder에서 폴더를 열어보면 된다.

**막혔을 때**:
- "claude: command not found" → Claude Code가 설치되지 않음. 운영진에게 설치 방법 요청
- 로그인 창이 뜨면 → claude.ai 계정으로 로그인
- /init 후 파일이 안 보이면 → 같은 폴더에서 실행했는지 확인. `pwd` 입력하면 현재 위치가 나옴
