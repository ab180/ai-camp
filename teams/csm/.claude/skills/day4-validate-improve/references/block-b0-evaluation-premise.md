## EXPLAIN

### 만든 사람은 결과를 편하게 본다

Day 3에서 각자 리캡 메일·문의 답변·미팅 노트 요약 스킬을 만들었다.
지금 그 결과를 내가 읽으면 거의 다 그럴듯해 보인다. 왜?

```
[문제]
내가 쓴 문장이 내 머릿속에서 재생될 때는 맥락이 이미 붙어 있다.
고객이 처음 읽을 때의 공백을 나는 느끼지 못한다.

[해법]
"다른 관점"이 같은 결과를 본다.
까다로운 고객 역할을 한 에이전트가 내 초안을 읽고
빠진 액션·오해 소지·톤을 집어준다.
```

### 오늘 B1은 운영진 시연 + 참가자 미니 실습

- 운영진이 "까다로운 고객" 에이전트를 시연
- 참가자는 자기 결과물 1건을 직접 넣고 피드백만 읽으면 됨

## EXECUTE

지금 어제 만든 스킬을 한 번 호출해보세요.

```
/skill [내 스킬 이름]
```

결과를 보면서: "이게 고객이 처음 받으면 어떻게 보일까?" 한 번 상상해보세요.
그 감각이 B1에서 까다로운 고객 피드백과 비교할 기준이 됩니다.
