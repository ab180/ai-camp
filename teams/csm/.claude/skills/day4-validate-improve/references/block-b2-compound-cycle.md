## EXPLAIN

### Compound 사이클 — Build → Use → Notice → Fix

```
1 · Build   일단 만든다 (완성도 60%면 된다. Day 3에서 한 게 Build)
2 · Use     실제로 쓴다 (진짜 고객·진짜 상황에. 예비 상황 X)
3 · Notice  불편함을 잡는다 (매번 수정하는 문장·반복 지적·누락 패턴 메모)
4 · Fix     rules에 기록한다 (CLAUDE.md·스킬·에이전트에 되먹인다)
```

첫 한 바퀴가 중요하지 않다. **매주 한 바퀴**가 중요하다.
3달 뒤 스킬은 1주차 버전과 완전히 다른 물건이 된다.

### 3번 반복 규칙

```
한 번 · 두 번은 우연일 수 있다.
세 번 반복되면 내 작업 루틴에서 구조적으로 빠지는 자리다.
그 자리를 스킬·rules에 추가한다.

× 흔한 함정                         ✓ Compound 방식
  "한국어로 답해줘" 매 세션 반복         지적 3번 → CLAUDE.md 한 줄
  피드백 받고도 스킬은 안 고침           스킬 템플릿 고정 섹션 추가
  5분 아끼려고 매번 같은 5분 씀          오늘 B1 피드백 → 다음부터 자동 적용
```

### 6축 O/X 자가진단

지난 4일 내 작업에 대해 O/X 체크:

| 축 | 질문 |
|----|------|
| **1 구조** | 내 CLAUDE.md에 담당 고객·톤·금지어가 적혀 있다 |
| **2 맥락** | MCP 1개 이상으로 SFDC·Notion·Gmail 등 실시간 데이터를 AI가 직접 읽는다 |
| **3 계획** | Deep Interview / Clarify로 "독자·질문·결론 순서"까지 구체화한 뒤 실행한다 |
| **4 실행** | 동작하는 스킬 1개 이상을 가지고 있고, 새 작업에 그 스킬을 호출한다 |
| **5 검증** | 내 초안을 "다른 관점" 에이전트에 한 번 넘겨 본 적이 있다 |
| **6 개선** | 피드백에서 3번 반복된 지적 1개를 CLAUDE.md·스킬에 되먹여 봤다 |

- **O가 4개 이상**: 충분한 첫 초안
- **X가 3개 이상**: 그중 한 축을 2주 프로젝트의 중심으로
