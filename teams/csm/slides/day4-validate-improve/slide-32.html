권한 관리 2/2 — Role 선택
CSM 팀 기본값은 Write
GitHub에는 5개 role이 있지만, 팀 스킬 repo에는 2~3개만 쓴다. 각자 권한을 너무 조이지 않는다.
1
Read
  • 보기만 가능
  • PR 생성 ×
  • 외부 옵저버용
2
Triage
  • 이슈 관리 가능
  • 코드 수정 ×
  • 잘 안 씀
3 · 권장
Write
  • 브랜치 생성 가능
  • PR 생성 가능
  • main 직접 푸시 ×
  • 팀원 전원 기본
4 · 팀장
Admin
  • 권한 관리 가능
  • main push · Settings
  • 팀장 1~2명만
Write = “브랜치 만들고 PR까지 자유”. main에 바로 push하지 못하는 것만 다르다. 리뷰 문화가 이 지점에서 시작된다.
