본인 스킬을 — 팀 자산으로
Day 2·3에서 PR 한 사이클은 이미 돌렸다. 이 부록은 팀 단위 운영에 필요한 추가 7장 — Play Store 모델, 권한, README 6축, 업로드, PR 워크플로우.
정의

팀 repo = 우리 팀 공용 스킬·rules·CLAUDE.md 저장소

한 사람이 만든 좋은 스킬을 다른 팀원이 즉시 가져다 쓸 수 있게 하는 것이 목적. 큰 룰만 정해두고 자유 등록(Play Store 모델). 팀 단위 Compound가 가능해진다.
왜 부록인가

오늘 본 콘텐츠는 본인 Compound가 우선

  • 본인 스킬에 Compound 파이프라인 만들기가 더 시급
  • 팀 repo 구조는 본인이 굴러가야 의미
  • 1주차 본인 사이클 → 2주차 팀 공유로 자연스러운 순서
언제 이 부록을 보면

1주차 끝, 또는 팀 누군가 PR 받기 시작할 때

  • 본인 스킬 1주 운영 후 팀에 공유하고 싶어질 때
  • 팀원 PR을 받았는데 README 양식이 없을 때
  • 신규 입사자에게 팀 자산 한 번에 넘기고 싶을 때
