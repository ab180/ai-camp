PR 워크플로우 — 왜 브랜치와 PR을 쓰나
main에 바로 박지 않는다
팀 repo는 공용 공간. 내 수정이 다른 사람의 스킬과 충돌할 수 있다. 브랜치 → PR → 리뷰 → merge 순서가 안전망이자 학습 루프.
× 이렇게 하면

main에 바로 push

  • 내 변경이 다른 스킬을 깬다
  • 팀원이 모르게 바뀜 → 놀람
  • 되돌리기 어렵다
  • “내가 뭘 바꿨지?” 기록이 사라진다
√ PR 워크플로우

브랜치 → PR → merge

  • 내 브랜치에서 자유롭게 실험
  • PR로 “이렇게 바꿔봤는데 봐줘”
  • 리뷰어가 1명 더 보니 검증된다
  • 변경 기록이 남는다 — Compound 자산
PR은 리뷰 받는 수단이 아니라 기록 · 커뮤니케이션 · 검증이 한 번에 되는 포맷. 축 5 검증의 팀 버전이다.
