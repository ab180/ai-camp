STEP 1 / 2
흐름 · 개인 → 팀
내 수정을 팀에 공유하는 5단계
처음 해보면 헷갈리지만 한 번만 흐름을 타면 다음부턴 익숙해진다. 5단계를 번호 순으로.
1
git clone <팀 repo URL>
처음 한 번만. 이후부터는 내 로컬에 이미 있음.
2
파일 수정 (내 CLAUDE.md 섹션 추가)
평소처럼 에디터에서 편집. Git이 아직 모름 — 저장만 해둔 상태.
3
git commit -m "feat: 담당 고객 B 추가"
수정을 스냅샷으로 박제. 메시지에 `<type>: <설명>` 형식 — feat · fix · docs 중.
4
git push
내 로컬 커밋을 Github에 업로드. 이제 팀이 볼 수 있음.
5
Github 웹 → PR 생성 → 팀장 리뷰 → merge
웹 UI에서 "Compare & pull request" 클릭. 리뷰 끝나면 merge.
3번 커밋 메시지는 <type>: <설명> 형식. feat(새 기능) · fix(수정) · docs(문서) · refactor(구조 개선) 중.
