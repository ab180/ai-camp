PR
PR = "내 수정을 합쳐줘"라는 쪽지
PR(Pull Request)은 Github 웹 UI에서 만든다. 터미널에서도 가능하지만 오늘은 웹 중심. 개념을 두 관점으로.
PR이 담는 것

무엇·왜·누가 볼지

무엇이 바뀌었나 — diff 자동 생성. 추가·삭제 줄 컬러 표시.
왜 바꿨나 — 설명 본문 작성. "고객 B 맥락을 추가했다" 한 줄이면 충분.
누가 리뷰할지 — Reviewers 지정. 짝꿍 or 팀장.
Merge까지 가는 길

리뷰 → 수정 → 승인 → merge

리뷰어가 코멘트를 남김 — 수정 요청 · 질문 · 승인.
수정 요청 있으면 같은 브랜치에 또 커밋하면 PR에 자동 반영.
승인되면 merge — 팀 repo에 반영 완료.
코드가 아니라 CLAUDE.md 문단 추가여도 같은 흐름이다. CSM 입장에서 PR은 "팀원에게 내 편집안 검토 부탁"하는 공식 쪽지.
