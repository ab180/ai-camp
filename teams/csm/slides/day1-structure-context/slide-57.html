내 규칙 / 팀 규칙 / 특정 폴더 규칙 — 3층으로 나눠 쓴다

혼자 쓸 때는 한 파일로 충분. 팀이 함께 쓰면 내 취향과 팀 약속이 섞여 부딪친다. Claude Code는 세 단계로 자동 분리해 충돌을 막는다.

내 규칙

나 혼자만 쓰는 규칙

~/.claude/CLAUDE.md

내 모든 프로젝트에 적용

팀 규칙

같이 일하는 사람 모두에게

프로젝트/CLAUDE.md · Git 공유

이 프로젝트 팀원 전체에 적용

폴더 규칙

특정 폴더에서만

프로젝트/특정폴더/CLAUDE.md

이 폴더 작업 시에만 적용

내 규칙

~/.claude/CLAUDE.md

적용 범위

내 모든 프로젝트에 공통. 팀원에게 공유되지 않는다.

예시

"이모지 안 씀", "한글 주석 선호"

팀 규칙

프로젝트/CLAUDE.md

적용 범위

이 프로젝트를 작업하는 모두에게. Git으로 팀 공유.

예시

"이 프로젝트는 snake_case", "테스트 80% 이상"

폴더 규칙

폴더/CLAUDE.md

적용 범위

이 폴더에서 작업할 때만 자동으로 로드.

예시

customers/acme/CLAUDE.md — ACME 고객 폴더에서만

덮어쓰기

좁은 범위가 넓은 범위를 이긴다

예) 내 규칙 "camelCase" / 팀 규칙 "snake_case" → 이 프로젝트는 snake_case 적용

3층 구조가 개인 취향과 팀 약속의 충돌을 막는다. 오늘 만든 CLAUDE.md는 팀 규칙 계층이라고 생각하면 된다.
