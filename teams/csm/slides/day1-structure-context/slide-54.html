.claude/rules/ — 주제별로 나눠둔다
어떤 규칙은 매번 필요하지 않다 — 고객 A 작업할 때만, 계약서 검토할 때만 필요하다. 그런 규칙을 주제별 작은 파일로 나눠두면 AI가 해당 상황에서만 자동으로 꺼내 읽는다.
① 어떻게 동작하나

파일 위에 "언제 쓸지" 적어둔다

규칙 파일 맨 위에 '언제 쓸지'를 적는다. 기술 용어로 glob — '파일 경로 패턴' 정도로 이해.

# .claude/rules/security.md glob: **/*.sql, **/auth/** └ SQL 파일이나 auth/ 폴더에서만 로드 - .env 파일 직접 수정 금지 - SQL은 parameterized query만
② 왜 좋은가

해당 상황에서만 꺼내 읽는다

AI는 auth 폴더나 SQL 파일을 건드릴 때만 이 규칙을 읽는다. 다른 작업에선 안 읽는다 — CLAUDE.md가 가벼워진다.

매번 필요한 규칙은 CLAUDE.md에, 가끔 필요한 규칙은 rules/에.

③ CSM 예시

고객·시즌·문서 종류로 나누기

  • rules/customer-acme.md ACME 폴더 작업할 때만 — 담당자 이름, 내부 용어, 특수 맥락
  • rules/quarterly-review.md QBR 폴더 작업할 때만 — QBR 작성 톤, 반드시 넣을 섹션
  • rules/legal-safe.md 계약서 폴더 작업할 때만 — 법적 검토 체크리스트
CLAUDE.md에는 "rules/에 이런 것들 있음" 한 줄만 써두면 된다. 내용은 각 파일에서 알아서.
