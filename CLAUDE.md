# Part 2 · 이메일 답변 작성 원칙 (사내 표준)

출처: AB180 CSM 사내 교육 — `https://www.notion.so/ab180/34dfbb0f3c244f0485fee6ac3f215af9`
모든 고객 문의 답변 초안은 아래 원칙을 따른다.

## 프로세스

0. **메일 수신 확인 + ETA 함께 회신** — 답변까지 시간이 걸리면 즉시 "수신함, 언제까지 회신" 짧게라도.
1. **핵심 이슈 파악** — 두서없는 메일에서 핵심부터 짚는다. 모호하면 **되묻는다** (추측 금지).
2. **원인·해결 방안 찾기** — 바로 가능하면 작성. 어려우면 가설을 **MECE**하게 정리하고 Reference/Escalation으로 검증.
3. **회신** — 정리된 원인·해결 방안 전달.

## Tips 11원칙

1. **결론(해결 방안) → 원인** 순서. 두괄식. Action Item 먼저.
2. **우선순위 분류**: 치명적 오류(리포트/차트 미로드, 데이터 누락)·캠페인 = 즉시. 대행사 자기 사정/핑프/일반은 SLA 24~48h 안 OK.
3. **가독성**: 주제별로 나누기(굵게/밑줄/큰 글씨). 단일 이슈 → `해결 방안 → 이슈 원인` 순.
4. **짧고 간결**. 의미없는 수식어 X. 가이드 문서는 핵심만 본문에, 자세한 건 **링크**.
5. **프레임워크**: `Why / What / How / If` 또는 `성과 / 질문 / 핵심이슈 / 원리 / 답 / 사례`.
6. 무엇을 묻는지 명확하지 않으면 **다시 물어본다**.
7. **내 잘못이 아니더라도 사과한다.**
8. 실수했으면 **빠르게 인정·정정** + 재발 방지 장치.
9. **"이메일 전달자"가 되지 말자.** "Braze 측에 확인…" 식으로 떠넘기지 않는다.
10. **ETA**: best는 (a) 수신 확인 + (b) 내가 이해한 Request + (c) 회신 ETA. 최소 (a)만이라도.
11. **용어 통일** — 한글/영어/대소문자. ("커스텀 이벤트" 한 형태로 고정)

## 핵심 3줄 요약 (준영님)

1. 상대가 "왜 연락이 안 되지?" 답답함을 느끼게 X.
2. 늦어질 듯하면 **ETA**.
3. **묻는 것에 답한다.** (핵심 이슈, 결론부터)

## 기타

- 컨텍스트 파악 — A 묻는데 B 답 X
- 두괄식 + 간결
- 메일로 설명 어려운 방대한 건 **컨콜 + 리캡 메일**도 옵션

## 초안 생성 시 자기 점검 체크리스트

- [ ] 즉시 답변 못 나가면 **ETA 짧은 회신** 버전도 같이 제안했는가
- [ ] **결론(=답)이 가장 위**에 있는가
- [ ] "Braze 측에 확인 필요" 류 **떠넘김 표현**이 없는가
- [ ] 가이드 문서 **링크**로 본문 분량을 줄였는가
- [ ] 용어가 본문 내에서 **통일**되었는가
- [ ] 모호한 부분은 추측 X, **되묻기** 문구로 처리했는가

# Part 3 · 메일 문의 응대 스타일 가이드 (사수 메일 분석)

> 사수 Suyoung Choi · Soyoung Jang 실제 응대 메일에서 추출한 표면 스타일. Part 2 원칙과 함께 적용.
> 출처: [CSM 응대 패턴 플레이북](https://www.notion.so/ab180/CSM-2026-Q1-Q2-350a69a8250781ed91a7c425bb6142ef)

## 인사 / 맺음말 (정형)

| 위치 | 형식 |
|---|---|
| 도입 | "안녕하세요 [고객 이름]님, AB180 [본인 이름]입니다." |
| 주말 다음날 추가 | "주말 동안 날씨가 참 좋았는데, 편안하게 보내셨길 바랍니다 :)" |
| 마무리 | "감사합니다." → 줄바꿈 → "[본인 이름] 드림" |
| 서명 끝 (항상) | `*원활한 히스토리 관리를 위해 팀메일 (dl_cs_team@ab180.co) 참조 부탁드립니다.` |

## 본문 구조

- **번호 매김**: 문의가 여러 건이면 1./2./3. 으로 분리해 항목별로 답변
- **인용 후 답변**: `>` 로 고객 문장 인용 → 본인 답변 (특히 다건 문의에 효과적)
- **굵은 강조**: `*…*` 으로 핵심 권장사항·지표·결론 강조 (예: *우선 채널 비용 연동을 진행*)
- **링크 묶음**: 가이드 URL은 bullet으로 모아서 본문 분량 압축 (Part 2 원칙 4와 정합)
- **시각 자료**: 화면 스크린샷 첨부 — 특히 사고·누락 진단 메일에서 효과적

## 톤 / 이모지

- 따뜻함: `:)` 정도가 기본. `💪🏻` 격려는 가끔만
- 양해 구할 때: *"양해 부탁드립니다. (_ _)"*
- 공감 표현: 고객이 진척 보고하면 *"다행입니다 :)"* 한 줄로 인정

## 권장 / 거절 / 추가 확인 / 컨콜 제안 톤

자주 쓰는 4가지 정형 표현 — 상황에 맞춰 그대로 가져다 쓴다.

- **권장**: *"~를 권장 드립니다"* / *"~를 우선적으로 진행해 주시길 권장 드립니다"*
- **거절·미지원**: *"~지원이 어려운 점 양해 부탁드립니다 (_ _)"* / *"아쉽게도 현시점에서는 ~이 어려운 상황입니다"*
- **추가 확인 약속**: *"다시 한번 확인해본 뒤 추가 회신 드리도록 하겠습니다"* — 모르는 건 추측 금지, 검증 후 회신
- **컨콜 제안**: *"이해 안 가는 내용이 있다면 편히 말씀주세요. 컨콜 링크 전달드리겠습니다"*

Part 2 원칙과의 정합:
- 권장 → "결론·Action Item 먼저" (원칙 1)
- 거절 → "내 잘못이 아니더라도 사과·양해" (원칙 7)
- 추가 확인 → "ETA 명시 + 떠넘김 방지" (원칙 9·10)
- 컨콜 제안 → "메일로 어려운 건 컨콜 + 리캡 메일" (기타)
